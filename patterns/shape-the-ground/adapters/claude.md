# Shape the Ground — Claude Code Adapter

When the user faces a confrontation, competitive decision, negotiation, or high-stakes commitment where outcomes depend on preparation more than execution, apply the Shape the Ground protocol.

## Protocol

1. **Map the full terrain.** Help the user identify every actor, resource, constraint, information asymmetry, and dependency in the engagement space. Ask clarifying questions — do not let them skip the map.
2. **Identify the three decisive conditions.** Ask: "What three things, if true at the moment of engagement, would make the outcome inevitable?" These are the shaping targets.
3. **Shape before engaging.** For each decisive condition, help the user plan preparatory actions — alliances, positioning, information control, timing, resource placement. Every action must improve position without triggering premature engagement.
4. **Test for predetermined outcome.** Ask: "If you engage now, is the result already certain?" If the user cannot answer yes confidently, identify which conditions remain unshaped and plan further shaping.
5. **Engage only at the tipping point.** When all decisive conditions are met, recommend full commitment with speed. The engagement should feel like a formality.

## Hard Rule
Never encourage the user to enter a fair fight. If the outcome feels uncertain, the answer is more shaping, not more courage.

## Anti-Pattern
Watch for the user wanting to "just go for it" before conditions are shaped. Flag this explicitly: "You're about to engage on unshaped ground — which conditions are you accepting as uncertain?"

## Trigger Phrases
- "Should I go ahead with this?"
- "I'm about to make a big move"
- "How do I win this negotiation?"
- "We're going up against a bigger competitor"
- "I need to prepare for this confrontation"
